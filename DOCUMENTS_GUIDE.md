# Pearls Migrator — Documents Guide

Этот файл фиксирует смысловую модель документов. Он нужен, чтобы парсер, JSON, база данных, каталог и будущая модель обработки текста одинаково понимали, где заголовок, где тело, где футер и какие даты у документа существуют.

## Главная Идея

PDF не является просто лекцией с одним заголовком и одной датой. В одном документе могут быть:

- редакционная дата публикации на нашем сайте;
- историческая дата создания документа;
- дата публикации в серии `Жемчужины Мудрости`;
- тип документа: диктовка, лекция, проповедь, призыв, материал;
- несколько строк заголовка;
- тело документа;
- финальная сноска после черты с историческим контекстом.

Reviewed JSON в `data/parsed/` должен сохранять эти данные явно, даже если часть полей пока не используется на сайте.

## Типы Документов

Основные типы:

- `dictation` — диктовка;
- `lecture` — лекция;
- `sermon` — проповедь;
- `prayer` — призыв или молитвенный блок, если документ фактически начинается с призыва;
- `material` — другой материал, если тип нельзя надежно определить.

Тип обычно определяется по заголовкам и футеру:

- `Диктовка Эль Мории` → `dictation`;
- `Лекция Элизабет Клэр Профет` → `lecture`;
- `Проповедь ...` → `sermon`;
- `Открывающий призыв`, `Призыв` → начало тела, а не продолжение заголовка.

Если тип найден в футере точнее, чем в заголовке, футер считается важным источником для уточнения метаданных.

## Три Даты Документа

### 1. Дата публикации на нашем сайте

Это редакционная дата, по которой документ входит в календарь сайта. Обычно она стоит первой строкой в подзаголовке:

```text
Октябрь 2006
Ноябрь 2006
Март 2006
```

Эта дата нужна для основного каталога сайта: список документов по годам и месяцам публикации у нас.

Один большой документ может относиться сразу к нескольким месяцам. Поэтому в целевой модели нужно хранить не только одну дату, но и список месяцев публикации.

Рекомендуемые поля:

```json
{
  "sitePublication": {
    "label": "Октябрь 2006",
    "year": 2006,
    "month": 10,
    "months": ["2006-10"],
    "sortDate": "2006-10-01"
  }
}
```

Если документ покрывает несколько месяцев:

```json
{
  "sitePublication": {
    "label": "Октябрь - Ноябрь 2006",
    "year": 2006,
    "month": 10,
    "months": ["2006-10", "2006-11"],
    "sortDate": "2006-10-01"
  }
}
```

### 2. Дата создания или рождения документа

Это дата, когда диктовка была реально продиктована, лекция дана, проповедь произнесена или материал впервые создан.

Чаще всего она находится в футере после черты:

```text
____________________________________________________
Лекция Элизабет Клэр Профет ... была дана в понедельник, 4 января 1988 года, в Ройял Тетон Рэнч, Монтана.
```

Эта дата нужна для исторического контекста. Пользователь должен иметь возможность нажать на год рождения документа и увидеть все документы, созданные в этом году.

Рекомендуемые поля:

```json
{
  "creation": {
    "date": "1988-01-04",
    "year": 1988,
    "raw": "Лекция Элизабет Клэр Профет ... была дана в понедельник, 4 января 1988 года, в Ройял Тетон Рэнч, Монтана."
  }
}
```

Если точная дата не извлечена, но год понятен, сохраняем год и исходный текст:

```json
{
  "creation": {
    "date": null,
    "year": 1991,
    "raw": "Диктовка Сен-Жермена была дана ... в воскресенье, 11 августа 1991 года ..."
  }
}
```

### 3. Дата публикации в `Жемчужинах Мудрости`

Это дата публикации головной организацией в виде выпуска `Жемчужин Мудрости`. Обычно строка содержит том, номер, автора и дату:

```text
Том 43 №№ 49, 50 – Элизабет Клэр Профет – 3, 10 декабря 2000 г.
Том 35 № 58 – возлюбленный Архангел Чамуил – 29 ноября 1992 г.
```

Эта информация есть не везде, но если она найдена, ее нужно сохранять отдельно.

Рекомендуемые поля:

```json
{
  "pearlPublication": {
    "volume": 43,
    "issue": "49, 50",
    "date": "2000-12-03",
    "rawDate": "3, 10 декабря 2000 г.",
    "raw": "Том 43 №№ 49, 50 – Элизабет Клэр Профет – 3, 10 декабря 2000 г."
  }
}
```

Если в строке указано несколько дат, `date` хранит первую дату для сортировки, а `rawDate` сохраняет исходное значение полностью.

## Автор

Автор должен быть отдельным нормализованным полем. Пользователь должен иметь возможность нажать на автора и увидеть только документы этого автора.

Источник автора:

- заголовок: `Диктовка Эль Мории`, `Лекция Элизабет Клэр Профет`;
- строка `Том ... – автор – дата`;
- футер после черты;
- имя файла — только запасной источник, если текст не дал автора.

Рекомендуемые поля:

```json
{
  "author": {
    "name": "Эль Мория",
    "slug": "el-morya",
    "raw": "Диктовка Эль Мории"
  }
}
```

Текущее поле `speaker` можно сохранить на переходный период, но целевая модель должна использовать `author`.

## Заголовок, Тело И Футер

Документ нужно хранить не только как список `paragraphs`, но и как три смысловые части.

### Заголовок

Заголовок включает строки до начала тела:

- редакционный месяц сайта;
- тип и автора;
- название документа;
- дополнительные строки заголовка.

Но если строка означает начало самого текста, она уже не должна считаться заголовком.

Маркеры начала тела:

- `Открывающий призыв`;
- `Призыв`;
- `Молитва`;
- `Преамбула`;
- другие похожие строки, которые фактически начинают текст документа.

Такие строки нужно сохранить в теле, но можно дополнительно пометить как первый раздел тела.

### Тело

Тело — основной текст документа без заголовочных строк и без финальной сноски после черты.

Для чтения на сайте, скачиваний, поиска и RAG основным содержимым считается именно тело.

### Футер

Футер — финальная информация после длинной черты из `_`, `-` или похожего визуального разделителя.

Футер нельзя выбрасывать. В нем часто находится:

- историческая дата создания;
- автор;
- место события;
- точная формулировка типа документа;
- контекст конференции или события.

Футер нужно хранить отдельно и не смешивать с основным телом.

## Целевая JSON-Модель

Минимальная целевая форма:

```json
{
  "slug": "1988-01-04-mother-mb2",
  "title": "Жемчужины Мудрости",
  "documentTitle": "Благотворное воздействие макробиотического питания на здоровье",
  "documentType": "lecture",
  "author": {
    "name": "Элизабет Клэр Профет",
    "slug": "elizabeth-clare-prophet",
    "raw": "Лекция Элизабет Клэр Профет"
  },
  "sitePublication": {
    "label": "Ноябрь 2006",
    "year": 2006,
    "month": 11,
    "months": ["2006-11"],
    "sortDate": "2006-11-01"
  },
  "creation": {
    "date": "1988-01-04",
    "year": 1988,
    "raw": "Лекция Элизабет Клэр Профет ... была дана в понедельник, 4 января 1988 года, в Ройял Тетон Рэнч, Монтана."
  },
  "pearlPublication": {
    "volume": null,
    "issue": null,
    "date": null,
    "rawDate": null,
    "raw": null
  },
  "parts": {
    "header": [
      "Ноябрь 2006",
      "Лекция Элизабет Клэр Профет",
      "«Благотворное воздействие макробиотического питания на здоровье»"
    ],
    "body": [
      {
        "text": "Открывающий призыв"
      },
      {
        "text": "Возлюбленное могущественное Я ЕСМЬ Присутствие..."
      }
    ],
    "footer": [
      {
        "text": "Лекция Элизабет Клэр Профет ... была дана в понедельник, 4 января 1988 года..."
      }
    ]
  },
  "paragraphs": [
    {
      "text": "Открывающий призыв"
    },
    {
      "text": "Возлюбленное могущественное Я ЕСМЬ Присутствие..."
    }
  ],
  "sourcePdf": "pearls/2006/1988_01_04_Mother(mb2).pdf",
  "jsonPath": "data/parsed/2006/1988_01_04_Mother(mb2).json",
  "parsedAt": "2026-05-10T07:34:11.435Z"
}
```

`paragraphs` можно оставить как совместимое поле для текущего рендера. Новая логика должна постепенно переходить на `parts.body`.

## Правила Извлечения

Парсер должен идти в таком порядке:

1. Извлечь все строки и абзацы из PDF.
2. Найти редакционную дату сайта в первых строках.
3. Найти строки заголовка до первого явного маркера тела.
4. Найти финальный разделитель и отделить футер.
5. Из заголовка и футера определить `documentType`, `author`, `documentTitle`.
6. Из футера извлечь `creation`.
7. Из строки `Том ... № ...` извлечь `pearlPublication`, если она есть.
8. Сохранить исходные строки в `parts.header`, `parts.body`, `parts.footer`.

Если парсер не уверен, он должен сохранить `raw`-значение и поставить нормализованное поле в `null`, а не терять информацию.

## Каталог И Фильтры

Основной каталог сайта строится по `sitePublication.year` и `sitePublication.months`.

Дополнительные пользовательские переходы:

- нажать на автора → список документов этого автора;
- нажать на год создания → список документов, созданных в этом году;
- нажать на год или месяц публикации сайта → список документов за этот период;
- позже: фильтр по типу документа.

База данных является runtime index. JSON остается источником правды. Если данные в базе и JSON различаются, побеждает JSON, а базу нужно пересидить.

## Практический Приоритет

Ближайшая задача — не идеально распарсить все исторические случаи, а перестать терять важные части документа.

Приоритет полей:

1. Сохранять `parts.header`, `parts.body`, `parts.footer`.
2. Явно хранить `sitePublication`.
3. Явно хранить `creation.raw` и `creation.year/date`, если удалось.
4. Явно хранить `author`.
5. Извлекать `pearlPublication`, если есть строка `Том ... № ...`.
6. Нормализовать `documentType`.
