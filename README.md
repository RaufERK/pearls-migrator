# Pearls Migrator

Минималистичный TypeScript-проект для превращения PDF-файлов из `pearls/` в веб-страницы.

Сейчас подключён один тестовый файл:

`pearls/2006/1994_12_25_Morya.pdf`

## Что делает проект

- читает PDF через `pdfjs-dist`;
- извлекает текстовые элементы вместе с координатами на странице;
- определяет базовый layout страницы: `single-column` или `two-column`;
- для двухколоночных PDF читает сначала левую колонку, затем правую;
- группирует элементы в строки и абзацы;
- отдаёт результат как HTML-страницу через Express и Handlebars;
- умеет сохранить распарсенный результат в JSON.

## Команды

```bash
npm run dev
```

Открыть страницу:

`http://localhost:3000/pearls/2006/1994-12-25-morya`

JSON API:

`http://localhost:3000/api/pearls/2006/1994-12-25-morya`

Сохранить результат парсинга:

```bash
npm run parse
```

Файл появится тут:

`data/parsed/1994_12_25_Morya.json`

Проверка TypeScript:

```bash
npm run build
```

## Текущая схема парсинга

Главная идея: не полагаться на простой текстовый порядок из PDF, потому что двухколоночные страницы часто смешивают строки. Вместо этого парсер берёт координаты каждого текстового фрагмента.

Алгоритм MVP:

1. На каждой странице извлекаются `x`, `y`, ширина, высота и текст.
2. Если страница горизонтальная и между текстовыми блоками есть крупный разрыв по `x`, она считается двухколоночной.
3. Для двух колонок текст разбивается по вертикальной линии раздела.
4. Внутри колонки элементы сортируются сверху вниз и слева направо.
5. Близкие по `y` элементы объединяются в строки.
6. Строки объединяются в абзацы по увеличенному вертикальному отступу или отступу первой строки.
