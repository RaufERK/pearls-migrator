# Миграция Концепции: Word-Брошюры Вместо PDF

Этот документ фиксирует новую рабочую концепцию архива. Он не удаляет и не заменяет старую модель из `DOCUMENTS_GUIDE.md`, а описывает следующий смысловой поворот перед будущей переработкой парсера.

## Главная Идея

Физическим источником контента становятся квартальные наборы Word-брошюр. Одна брошюра соответствует одному месячному выпуску `Жемчужин Мудрости` на сайте.

PDF больше не считается основным источником для парсинга. Папка `pearls-pdf` остается архивом исходных PDF-файлов и может использоваться позже только как справочная ссылка на оригинальный файл, если такая связь нужна на сайте.

Новая модель сохраняет идею `Жемчужины` как физического контейнера: внутри одной месячной брошюры может быть один или несколько материалов. Отличие от старого подхода только в источнике: вместо PDF парсим более чистый Word-файл.

Отдельные файлы из папок `Оригиналы` остаются справочным и редакторским источником, но не являются главным входом для MVP-парсинга.

## Источник Данных

Основная папка источников:

```text
pearls-word/
```

Целевая организация источников:

```text
pearls-word/
  2025/
    1-й квартал/
      Брошюры/
        ЖМ 1_кв.1_январь 2025_....docx
        ЖМ 2_кв.1_февраль 2025_....docx
        ЖМ 3_кв.1_март 2025_....docx
      Оригиналы/
        ...
```

Главная папка для парсинга:

```text
pearls-word/<год>/<квартал>/Брошюры/
```

В одном квартале ожидается три месячных брошюры. Парсер должен идти квартал за кварталом, чтобы результат можно было спокойно проверять руками.

## Что Парсим

Парсер должен обрабатывать русские Word-брошюры из папок `Брошюры`.

Рабочий формат парсера — `.docx`.

Если исходник лежит в старом формате `.doc`, его нужно автоматически конвертировать в `.docx` перед парсингом. Сам парсер не должен пытаться напрямую читать `.doc`, потому что это старый бинарный формат Word и он менее стабилен для извлечения текста.

На первом этапе не используем как основной источник:

- PDF-файлы из `pearls-pdf`;
- отдельные файлы из `Оригиналы`;
- временные Word-файлы вида `~$...`;
- английские оригиналы.

## Конвертация `.doc` В `.docx`

Автоматическая конвертация делается через LibreOffice в headless-режиме:

```bash
soffice --headless --convert-to docx --outdir tmp/converted "path/to/file.doc"
```

На macOS бинарь может быть доступен как `soffice`, либо по полному пути:

```bash
/Applications/LibreOffice.app/Contents/MacOS/soffice --headless --convert-to docx --outdir tmp/converted "path/to/file.doc"
```

В текущей рабочей среде LibreOffice уже установлен и доступен двумя способами:

```text
/usr/local/bin/soffice
/Applications/LibreOffice.app/Contents/MacOS/soffice
```

Проверенная версия:

```text
LibreOffice 26.2.3.2
```

Поэтому будущий CLI может сначала вызывать `soffice` из `PATH`, а если команда недоступна, пробовать macOS fallback-путь `/Applications/LibreOffice.app/Contents/MacOS/soffice`.

Будущий CLI должен делать это автоматически:

1. Найти `.doc` или `.docx` в папке `Брошюры`.
2. Если файл уже `.docx`, сразу отдать его Word-парсеру.
3. Если файл `.doc`, создать временную папку вроде `tmp/converted/<hash-or-year-quarter>/`.
4. Вызвать `soffice --headless --convert-to docx --outdir <tmp>` для этого файла.
5. Проверить, что рядом появился `.docx`.
6. Передать сконвертированный `.docx` в парсер.
7. Игнорировать временные Word-файлы `~$...`.

Конвертация должна быть техническим подготовительным шагом. JSON должен хранить путь к исходному файлу из `pearls-word`, а при необходимости дополнительно путь к временно сконвертированному `.docx` только для диагностики.

## Сущности

### Материал

`Материал` остается смысловой единицей внутри жемчужины.

Одна брошюра может содержать один или несколько материалов:

- диктовке;
- лекции;
- проповеди;
- учению;
- молитвенному или другому текстовому материалу.

Материал хранится внутри `documents[]` с собственными `header`, `body`, `footer`, автором, датами и типом.

### Жемчужина

`Жемчужина` снова считается физическим контейнером и основным JSON-документом.

Одна Word-брошюра из папки `Брошюры` соответствует одной жемчужине и одному JSON-файлу в `data/parsed/`.

Внутри жемчужины сохраняется список `documents[]`. Это хорошо совпадает с текущей архитектурой сайта и уже отработанной моделью PDF-парсинга.

### Публикационный Блок

Для группировки жемчужин нужна логика публикации:

- год публикации на сайте;
- квартал публикации на сайте;
- месяц публикации на сайте, если известен;
- порядок материала внутри месяца или квартала;
- необязательная метка выпуска.

Эта группировка должна помогать показывать три месячных выпуска внутри квартала, но не должна ломать внутреннюю структуру `documents[]`.

## JSON-Модель

Каждая Word-брошюра парсится в один JSON.

Целевая форма близка к текущей PDF-модели, но источник становится Word:

```json
{
  "slug": "2025-03",
  "title": "Жемчужины Мудрости",
  "sourceWord": "pearls-word/2025/1-й квартал/Брошюры/ЖМ 3_кв. 1_март 2025_....docx",
  "archivePdf": null,
  "sitePublication": {
    "year": 2025,
    "quarter": 1,
    "month": 3,
    "sortDate": "2025-03-01"
  },
  "documentsCount": 3,
  "documents": [
    {
      "documentTitle": null,
      "documentType": "dictation",
      "author": {
        "name": "Архангел Гавриил",
        "slug": "archangel-gabriel",
        "raw": "Диктовка Архангела Гавриила"
      },
      "creation": {
        "date": null,
        "year": null,
        "raw": null
      },
      "pearlPublication": {
        "volume": null,
        "issue": null,
        "date": null,
        "rawDate": null,
        "raw": null
      },
      "parts": {
        "header": [],
        "body": [],
        "footer": []
      }
    }
  ]
}
```

Массив `documents[]` остается основной моделью. Если в брошюре один материал, массив содержит один элемент. Если в брошюре несколько диктовок или лекций, каждая хранится отдельным внутренним документом.

## Ручная Проверка Метаданных

Поскольку Word-брошюры могут отличаться оформлением, парсер не должен пытаться идеально угадать все с первого раза.

Рабочий процесс:

1. Берется один квартал.
2. `.doc` автоматически конвертируется в `.docx`, если нужно.
3. Парсер генерирует JSON из трех месячных брошюр.
4. Редактор вручную проверяет названия, авторов, даты, типы и разбиение на `documents[]`.
5. Ошибки фиксируются либо правилами парсера, либо редакторскими override-метаданными.
6. После этого обрабатывается следующий квартал.

Важно: ручная проверка является частью процесса. Новая цель не в том, чтобы сразу автоматически распарсить весь архив, а в том, чтобы двигаться аккуратно и воспроизводимо.

## Отношение К Старой PDF-Модели

Старая модель описывала PDF как контейнер, внутри которого могло быть несколько материалов. Для исторических PDF это было правильно.

Новая модель меняет источник, но сохраняет контейнерную идею:

- раньше физический файл мог быть контейнером;
- теперь физический файл тоже остается контейнером, но это Word-брошюра вместо PDF;
- одна брошюра соответствует одному месячному выпуску;
- внутренние материалы по-прежнему живут в `documents[]`.

Старый документ `DOCUMENTS_GUIDE.md` остается полезным для правил заголовка, тела, футера, дат, автора, типа документа и разбиения контейнера на внутренние материалы. Будущий парсер должен применять эти правила к Word-брошюре.

## Будущий План Переработки

Когда начнется реализация, изменения стоит делать отдельными шагами:

1. Добавить Word-first парсер для `.docx`.
2. Добавить подготовительный шаг конвертации `.doc` в `.docx` через LibreOffice headless.
3. Научить CLI читать `pearls-word/<год>/<квартал>/Брошюры`, а не `pearls` или `pearls-pdf`.
4. Переименовать PDF-зависимые поля в нейтральные: `sourcePdf` -> `sourceWord` или `sourcePath`.
5. Убрать из новой генерации PDF-специфику: страницы, колонки, layout, `pdf-processing-map`.
6. Сохранить текущую модель `documents[]` для внутренних материалов брошюры.
7. Обновить сидинг базы и каталог под Word-источник.
8. Только после этого решать, нужна ли связь с архивными PDF.

## Принятое Решение

Новая целевая концепция:

```text
одна русская Word-брошюра = одна жемчужина = один JSON
```

Если брошюра хранится в `.doc`, она автоматически конвертируется в `.docx`, и только после этого парсится.

`Жемчужина Мудрости` на сайте остается контейнером публикации, внутри которого могут находиться несколько отдельных материалов. Это ближе к текущей архитектуре проекта и использует уже отработанную модель разбиения на внутренние документы.
